# MY-UI: SwiftUI-inspired React Framework

MY-UI — это экспериментальный UI-фреймворк для React, написанный на TypeScript. Главная идея проекта — перенести концепцию цепочек вызовов (method chaining) и компоновки интерфейсов из Apple SwiftUI в мир веб-разработки.

Этот проект — поле для экспериментов. Он исследует, насколько реально и удобно собирать интерфейс в React не через привычные HTML-теги и JSX, а используя объектно-ориентированный, декларативный подход.

---

## 🎯 Задумка: к чему мы стремимся

В конечном счете, цель фреймворка — создать инструмент, который позволит:
1. Собирать UI как конструктор из готовых классов с предсказуемым поведением.
2. Кардинально снизить визуальный шум в коде: избавиться от бесконечных вложенных `<div>` и простыней `className`.
3. Сделать стилизацию интуитивной: чтобы автокомплит IDE сам подсказывал доступные модификаторы (`.padding()`, `.background()`, и т.д.).
4. Обеспечить строгую типизацию, минимизируя визуальные баги на этапе написания кода.

---

## ⚠️ Текущие проблемы и ограничения (Честно о главном)

На данный момент фреймворк находится на ранней стадии развития (Proof of Concept) и имеет ряд существенных архитектурных проблем:

* **Проблема производительности (Аллокация объектов):** Архитектура реализована «в лоб». При каждом рендере React-компонента на каждый "чих" заново создаются десятки и сотни новых экземпляров классов (`new MYText()`, `new MYVStack()` и всех их модификаторов). Это создает большую нагрузку на сборщик мусора (Garbage Collector) и может приводить к просадкам FPS в сложных интерфейсах.
* **Ограниченный функционал:** Реализованы только самые базовые компоненты и стеки. Не хватает многих вещей, необходимых для полноценного продакшена (сложные анимации, доступность/a11y, управление фокусом, виртуализация списков).
* **Стилизация через инлайн-стили:** Сейчас модификаторы преобразуются в `React.CSSProperties`. Это ограничивает использование медиа-запросов (media queries), псевдоклассов (например, `:hover`, `:focus`) и не так оптимально, как генерация статического CSS.
* **Конфликты с Flexbox/Grid:** Иногда поведение контейнеров (например, ZStack) требует ручной корректировки из-за особенностей того, как CSS Grid и Flexbox обрабатывают вложенные элементы в вебе.

## 📱 Примеры (Showcase)

Несмотря на ограничения, на фреймворке уже можно собирать рабочие интерфейсы. В проекте есть 3 примера:

### 1. Упрощенная копия калькулятора iOS (app/page.tsx)
Демонстрирует базовую стейт-логику калькулятора, сетку кнопок через вложенные стеки (MYHStack внутри MYVStack) и применение скруглений (капсулы и continuous corners). 

### 2. Макет музыкального плеера (Apple Music Style)
Показывает работу с простыми интервалами (через кастомные методы .onChange и .onDisappear), использование форм (MYCapsule для прогресс-бара) и позиционирование элементов друг над другом с помощью MYZStack.

### 3. CLOPE Mushroom Clustering
Интерактивный дашборд для визуализации алгоритма кластеризации на датасете с грибами. 
Демонстрирует, как можно разделять логику (ViewModels) и UI, использовать нативный скролл (MYScrollView) и бесшовно смешивать компоненты MY-UI с классическими React-элементами (через обертку MYAnyView для отрисовки стандартной таблицы).

## 📖 Пример синтаксиса MY-UI

Все UI-элементы наследуются от базового класса MYView. Для интеграции полученного дерева объектов в обычный React используется обертка <RenderMYView />.

import React from "react";
import { 
  MYVStack, 
  MYText, 
  MYColor, 
  RenderMYView, 
  MYRoundedRectangle 
} from "@/my-ui";

const myCardView = new MYVStack([
  new MYText("Hello, SwiftUI in React!")
    .font({ size: 24, weight: "bold" })
    .foregroundStyle(MYColor.white),
    
  new MYText("This is an experimental framework.")
    .font({ size: 16 })
    .foregroundStyle(MYColor.rgb(180, 180, 180))
], 12, "center")
  .padding({ edges: "all", length: 32 })
  .background(MYColor.rgb(31, 41, 55))
  .clipShape(new MYRoundedRectangle(24, "continuous"));

export default function App() {
  return (
    <div style={{ display: 'flex', justifyContent: 'center', padding: '50px' }}>
        <RenderMYView view={myCardView} />
    </div>
  );
}

## 🤝 Вклад в проект и Архитектура

Архитектура построена на паттерне "Декоратор". Каждый визуальный модификатор (например, .padding()) оборачивает исходный класс в новый объект MYModifiedContent. 

Если вы хотите поэкспериментировать вместе со мной:
1. Новые компоненты создаются в my-ui/components (наследуются от MYView, реализуют метод body()).
2. Модификаторы лежат в my-ui/modifiers (реализуют интерфейс MYViewModifier). Их также нужно прописать в ядре (core/View.ts) для поддержки чейнинга.
